# Plan 1 — Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Lay down the empty-but-correct repository skeleton so later plans can add backend, frontend, infrastructure, workflows, and demo content on a clean base.

**Architecture:** This plan creates repository-level conventions, documentation skeletons, azd metadata, Dependabot configuration, and empty subsystem directories only. It intentionally leaves application code, Bicep modules, workflows, azd service definitions, demo branches, and full demo documentation to later plans so each subsequent plan has a stable contract to build on.

**Tech Stack:** Git, GitHub repository metadata, Dependabot v2, Azure Developer CLI (`azd`) manifest YAML, Markdown, EditorConfig, MIT license.

---

## File Responsibility Map

- `.gitignore` keeps local build outputs, azd state, editor metadata, OS files, generated Bicep JSON, and local secret files out of Git.
- `.gitattributes` normalizes text files to LF and marks common image/archive formats as binary.
- `.editorconfig` establishes indentation, line endings, and whitespace conventions for future Java, TypeScript, YAML, Bicep, and Markdown files.
- `LICENSE` records the MIT license for owner Joran Bergfeld in 2026.
- `README.md` introduces the demo, links to the approved design, and marks implementation areas that later plans will expand.
- `.github/dependabot.yml` enables weekly dependency update groups for the future Maven backend, npm frontend, and GitHub Actions workflows.
- `.github/CODEOWNERS` assigns all files to `@JoranBergfeld`.
- `.github/PULL_REQUEST_TEMPLATE.md` standardizes PR context and demo-branch impact checks.
- `azure.yaml` establishes the azd project name and metadata while keeping services and hooks inactive until Plan 5.
- `.gitkeep` files preserve empty subsystem directories that later plans will populate.
- `docs/ARCHITECTURE.md` links to the approved design and reserves the architecture reference area for Plan 8 diagrams.
- `scripts/seed-vulnerabilities.md` reserves the seeded vulnerability inventory for Plan 7.

## What This Demo Proves

The final demo will show four moments:

1. Secret scanning push protection blocks a fake token before it reaches the remote repository.
2. CodeQL blocks a SQL injection pull request before it can merge into the secure branch.
3. Defender for Containers scans a vulnerable image in ACR and AKS admission control denies the deployment.
4. Defender for Cloud correlates GHAS findings with the running AKS workload.

## Architecture

The target architecture uses:

- React 18 + TypeScript + Vite on Azure Static Web Apps.
- Spring Boot 3.3.x with Java 21 on AKS.
- Azure Database for PostgreSQL Flexible Server on private networking.
- Azure Container Registry Premium with private endpoint.
- Azure Key Vault with RBAC authorization.
- GitHub Actions with OpenID Connect federated credentials.
- Defender for Cloud plans for CSPM, Containers, Key Vault, Open-Source Relational Databases, and Resource Manager.

See [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) for the architecture reference as it is expanded.
