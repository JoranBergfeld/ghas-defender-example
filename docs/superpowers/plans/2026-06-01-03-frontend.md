# Plan 3 — Frontend Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a clean, runnable Vite + React 18 + TypeScript 5 SPA in `src/frontend/` that calls the Plan 2 backend and passes Vitest unit tests without seeding any frontend vulnerability.

**Architecture:** The frontend is a small React SPA with route-level components for item listing, search, and login. Typed API wrapper modules sit behind one axios client configured by `VITE_API_BASE_URL`, and tests mock the network boundary so CI can run without a backend. Search results are rendered as JSX text so React performs default escaping; Plan 7 can later seed the XSS in `SearchResults.tsx` only on the `vulnerable` branch.

**Tech Stack:** Vite 5, React 18, TypeScript 5, axios, react-router-dom, Vitest with jsdom, React Testing Library, ESLint, Prettier, Azure Static Web Apps config.

---

## Cross-plan assumptions

- Plan 1 has created `src/frontend/.gitkeep` and repository ignore rules for Node artifacts, including `node_modules/`, `dist/`, `coverage/`, and `.env*`.
- Plan 2 exposes `http://localhost:8080` with these shared contracts:
  - `GET /api/items` returns `Item[]`.
  - `POST /api/items` with bearer auth returns `Item`.
  - `PUT /api/items/{id}` with bearer auth returns `Item`.
  - `DELETE /api/items/{id}` with bearer auth returns `204`.
  - `GET /api/items/search?q=<query>` returns `Item[]`.
  - `POST /api/auth/login` with `{ username, password }` returns `{ token }`.
- Branches remain `main`, `secure`, and `vulnerable`; all commits use Conventional Commits plus the Copilot co-author trailer.
- This plan intentionally implements the secure frontend only. It must not introduce `dangerouslySetInnerHTML`.

## File responsibility map

- `src/frontend/package.json` declares scripts, runtime dependencies, and development dependencies.
- `src/frontend/package-lock.json` is generated by `npm install` and committed for `npm ci` in CI.
- `src/frontend/vite.config.ts` configures Vite and Vitest in one file.
- `src/frontend/tsconfig.json` and `src/frontend/tsconfig.node.json` configure TypeScript for app, tests, and Vite config.
- `src/frontend/.eslintrc.cjs`, `src/frontend/.eslintignore`, and `src/frontend/.prettierrc` configure linting and formatting.
- `src/frontend/index.html`, `src/frontend/src/main.tsx`, and `src/frontend/src/index.css` are the browser entry point and styling.
- `src/frontend/src/App.tsx` owns routing: `/`, `/search`, and `/login`.
- `src/frontend/src/components/ItemList.tsx` fetches and renders `GET /api/items`.
- `src/frontend/src/components/SearchResults.tsx` reads `?q=`, fetches search results, and safely renders result text with React escaping.
- `src/frontend/src/components/LoginForm.tsx` posts credentials and stores the JWT token under `auth_token`.
- `src/frontend/src/api/client.ts` exports the axios instance and bearer-token interceptor.
- `src/frontend/src/api/items.ts` and `src/frontend/src/api/auth.ts` export typed API functions.
- `src/frontend/src/types.ts` defines shared frontend types matching Plan 2.
- `src/frontend/src/test/setup.ts` installs jest-dom matchers for Vitest.
- `src/frontend/src/__tests__/` contains route, component, and API tests.
- `src/frontend/.env.example` documents `VITE_API_BASE_URL`.
- `src/frontend/.env.development` provides the local default and remains ignored by git.
- `src/frontend/staticwebapp.config.json` configures SPA fallback and security headers for Azure Static Web Apps.

import axios, { AxiosHeaders } from 'axios';

export const AUTH_TOKEN_STORAGE_KEY = 'auth_token';

export const API_BASE_URL = import.meta.env.VITE_API_BASE_URL ?? 'http://localhost:8080';

export const apiClient = axios.create({
  baseURL: API_BASE_URL,
  headers: { 'Content-Type': 'application/json' },
});

apiClient.interceptors.request.use((config) => {
  const token =
    typeof window === 'undefined' ? null : window.localStorage.getItem(AUTH_TOKEN_STORAGE_KEY);

  if (token) {
    const headers = AxiosHeaders.from(config.headers);
    headers.set('Authorization', `Bearer ${token}`);
    config.headers = headers;
  }

  return config;
});
TS

import { FormEvent, useState } from 'react';
import { login } from '../api/auth';
import { AUTH_TOKEN_STORAGE_KEY } from '../api/client';

export default function LoginForm() {
  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');
  const [isSubmitting, setIsSubmitting] = useState(false);
  const [message, setMessage] = useState<string | null>(null);
  const [error, setError] = useState<string | null>(null);

  async function handleSubmit(event: FormEvent<HTMLFormElement>) {
    event.preventDefault();
    setIsSubmitting(true);
    setMessage(null);
    setError(null);

    try {
      const response = await login({ username, password });
      window.localStorage.setItem(AUTH_TOKEN_STORAGE_KEY, response.token);
      setMessage('Signed in successfully.');
      setPassword('');
    } catch {
      setError('Sign in failed. Check the username and password.');
    } finally {
      setIsSubmitting(false);
    }
  }

  return (
    <section aria-labelledby="login-heading" className="page-section narrow">
      <div className="section-heading">
        <p className="eyebrow">Admin</p>
        <h1 id="login-heading">Sign in</h1>
      </div>

      <form className="form-card" onSubmit={handleSubmit}>
        <label htmlFor="username">Username</label>
        <input
          autoComplete="username"
          id="username"
          name="username"
          onChange={(event) => setUsername(event.target.value)}
          required
          type="text"
          value={username}
        />

        <label htmlFor="password">Password</label>
        <input
          autoComplete="current-password"
          id="password"
          name="password"
          onChange={(event) => setPassword(event.target.value)}
          required
          type="password"
          value={password}
        />

        <button disabled={isSubmitting} type="submit">
          {isSubmitting ? 'Signing in...' : 'Sign in'}
        </button>
      </form>

      {message ? <p role="status">{message}</p> : null}
      {error ? <p role="alert">{error}</p> : null}
    </section>
  );
}
